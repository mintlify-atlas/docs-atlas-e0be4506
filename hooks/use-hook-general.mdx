## Overview

The `useHook_General` hook is a convenience hook that aggregates multiple custom hooks into a single, unified interface. This pattern simplifies component code by providing a single import point for commonly used hooks.

## Source Code

```typescript hooks/useHook.ts
import { useModulos } from "./useModulos";

export const useHook_General = () => {
  const { modulosOptions } = useModulos();
  return { modulosOptions };
};
```

## Purpose and Benefits

### Aggregation Pattern

The hook uses the aggregation pattern to:

1. **Simplify imports**: Import one hook instead of multiple
2. **Centralize hook composition**: Manage hook dependencies in one place
3. **Provide consistency**: Ensure components use the same hook combinations
4. **Enable scalability**: Easy to add new hooks to the interface

### When to Use

Use `useHook_General` when:
- You need access to commonly used hooks
- You want to reduce import statements in components
- Multiple components need the same hook combination

## Return Values

<ResponseField name="modulosOptions" type="ModuloOption[]">
  Array of module objects formatted for PrimeReact Dropdown components. Returned from the `useModulos` hook.
  
  Each option contains:
  - `id` (number): Module identifier
  - `nombre` (string): Module name
  - `label` (string): Display label for dropdown
  - `value` (number): Selection value for dropdown

## How It Works

### Hook Aggregation

The hook internally calls other custom hooks and returns their values:

```tsx
// Inside useHook_General
const { modulosOptions } = useModulos();  // Call useModulos
return { modulosOptions };                 // Return its values
```

This creates a single point of access for multiple hook functionalities.

### Future Extensions

The pattern allows easy extension:

```tsx
// Example: Adding more hooks
import { useModulos } from "./useModulos";
import { useMotivos } from "./useMotivos";
import { useEconomicos } from "./useEconomicos";

export const useHook_General = () => {
  const { modulosOptions } = useModulos();
  const { motivosOptions } = useMotivos();
  const { economicosData } = useEconomicos();
  
  return { 
    modulosOptions,
    motivosOptions,
    economicosData 
  };
};
```

## Usage Examples

### Basic Usage

```tsx
import { useHook_General } from '../hooks/useHook';
import { Dropdown } from 'primereact/dropdown';
import { useState } from 'react';

export const MyComponent = () => {
  const { modulosOptions } = useHook_General();
  const [selected, setSelected] = useState(null);
  
  return (
    <Dropdown
      value={selected}
      onChange={(e) => setSelected(e.value)}
      options={modulosOptions}
    />
  );
};
```

## Advantages Over Direct Hook Usage

### Without useHook_General

```tsx
import { useModulos } from '../hooks/useModulos';
import { useMotivos } from '../hooks/useMotivos';
import { useEconomicos } from '../hooks/useEconomicos';

export const MyComponent = () => {
  const { modulosOptions } = useModulos();
  const { motivosOptions } = useMotivos();
  const { economicosData } = useEconomicos();
  
  // Component logic...
};
```

### With useHook_General

```tsx
import { useHook_General } from '../hooks/useHook';

export const MyComponent = () => {
  const { modulosOptions, motivosOptions, economicosData } = useHook_General();
  
  // Component logic...
};
```

## Design Pattern

This follows the **Facade Pattern** in software design:

- **Simplifies interface**: Provides a simple interface to complex subsystems
- **Reduces coupling**: Components don't need to know about individual hooks
- **Centralizes changes**: Modifications to hook composition happen in one place
- **Improves maintainability**: Easier to refactor hook dependencies
