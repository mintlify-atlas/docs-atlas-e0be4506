## Styling Architecture Overview

SUGO Front uses a multi-layered styling approach that combines:

- **PrimeReact** theme system for component styling
- **Bootstrap** utilities for layout and spacing
- **Custom CSS** for application-specific styles
- **Inline styles** for dynamic component customization

## Import Structure

All styling dependencies are imported in a specific order in `main.tsx` to ensure proper CSS cascading:

```tsx src/main.tsx
import "bootstrap/dist/css/bootstrap.min.css";
import "bootstrap-icons/font/bootstrap-icons.css";
import "primereact/resources/themes/saga-blue/theme.css";
import "primereact/resources/primereact.min.css";
import "primeicons/primeicons.css";
import "./App.css";
```

<Note>
The import order matters! Bootstrap is loaded first, followed by PrimeReact themes, and finally custom CSS to allow overrides.
</Note>

## Inline Styles Usage

For dynamic or component-specific styling, inline styles are used:

```tsx src/components/Header.tsx
<div style={{ display: "flex", alignItems: "center", gap: "1.5rem" }}>
  <i className="pi pi-user" style={{ fontSize: "1.1rem" }}></i>
  
  <button
    style={{
      background: "none",
      border: "none",
      cursor: "pointer",
      padding: 0,
      marginRight: "0.5rem",
    }}
  >
    <i className="pi pi-sign-out" style={{ fontSize: "1.2rem", color: "#4caf50" }}></i>
  </button>
</div>
```

## Styling Best Practices

<Steps>
  <Step title="Use PrimeReact components first">
    Leverage built-in PrimeReact components and their styling system before creating custom solutions.
  </Step>
  
  <Step title="Apply Bootstrap utilities for layout">
    Use Bootstrap's flexbox utilities (`d-flex`, `justify-content-center`, etc.) for responsive layouts.
  </Step>
  
  <Step title="Override in App.css">
    Place component overrides and custom styles in `App.css` with specific class names.
  </Step>
  
  <Step title="Use inline styles sparingly">
    Reserve inline styles for dynamic values or one-off component-specific adjustments.
  </Step>
</Steps>

<Warning>
When overriding PrimeReact styles, you may need to use `!important` to ensure your custom styles take precedence.
</Warning>
