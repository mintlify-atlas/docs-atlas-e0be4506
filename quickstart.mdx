## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js 24.14.0 or higher** (Alpine-based)
- **npm**, **yarn**, or **pnpm** package manager
- **Git** for cloning the repository

<Note>
  SUGO Front is tested with Node.js 24.14.0-alpine. While other versions may work, this is the recommended version.
</Note>

## Quick Installation

<Steps>
  <Step title="Clone the repository">
    Clone the SUGO Front repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd sugo-front
    ```
  </Step>
  
  <Step title="Install dependencies">
    Install the required packages using your preferred package manager:
    
    <CodeGroup>
      ```bash npm
      npm install
      ```
      
      ```bash yarn
      yarn install
      ```
      
      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
    
    <Info>
      The project includes a `preinstall` script that runs automatically before installation.
    </Info>
  </Step>
  
  <Step title="Start the development server">
    Launch the Vite development server with hot module replacement:
    
    <CodeGroup>
      ```bash npm
      npm run dev
      ```
      
      ```bash yarn
      yarn dev
      ```
      
      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>
    
    The server will start on port **5173** with polling enabled for file watching.
  </Step>
  
  <Step title="Access the application">
    Open your browser and navigate to:
    
    ```
    http://localhost:5173
    ```
    
    You should see the SUGO main page. Use the navigation to access:
    - **Dispatch Form**: `http://localhost:5173/despacho`
    - **Reception Form**: `http://localhost:5173/recepcion`
  </Step>
</Steps>

## Application Routes

SUGO Front includes the following routes:

| Route | Component | Description |
|-------|-----------|-------------|
| `/` | `Sugo_main` | Home page with SUGO branding |
| `/despacho` | `FormularioDespacho` | Dispatch form with module, economic, and reason fields |
| `/recepcion` | `FormularioRecepcion` | Reception form interface |

## Development Configuration

The Vite configuration in `vite.config.ts` provides:

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
  server: {
    host: true,
    port: 5173,
    watch: {
      usePolling: true,
    },
  },
});
```

<Info>
  **Polling is enabled** for file watching, which is useful for Docker environments and certain file systems.
</Info>

## First Steps

Once the application is running:

1. **Explore the Dispatch Form** at `/despacho`:
   - Select a module from the dropdown (Oficinas centrales, Modulo 1-7)
   - Enter vehicle economic number
   - Select dispatch reason
   - Submit the form

2. **Navigate using the Header** component which provides consistent navigation across routes

3. **Check the browser console** for any React or application-specific messages

## Example: Using the Dispatch Form

The dispatch form uses PrimeReact components:

```tsx
import { Dropdown } from "primereact/dropdown";
import { InputText } from "primereact/inputtext";
import { Button } from "primereact/button";

// Module selection
<Dropdown
  inputId="dd-city"
  value={selectModulo}
  onChange={(e) => setSelectModulo(e.value)}
  options={modulosOptions}
  className="select"
/>

// Submit button
<Button 
  icon="pi pi-check" 
  label="Enviar" 
  severity="success" 
/>
```
